REPLIT VOLUME DISPLAY FIX
=========================
MarshallComics.com — May 27, 2026

THE PROBLEM (visible in screenshots):
--------------------------------------
The "Every Book" view is showing "All-New Ultimates (Fiffe)" as a SEPARATE title 
from "All-New Ultimates", because the app is appending volume or writer data into 
the title field at the display layer.

Michel Fiffe wrote All-New Ultimates Vol 1. He should NOT appear in the title string.
The title is "All-New Ultimates" — Vol 1 — for all issues.

THE FIX — 3 changes in Replit:
-------------------------------

1. TITLE DISPLAY — never append writer name to title
   In your book list component, the title field should ONLY display comic.title
   Do NOT concatenate comic.writer or any other field into the title display.
   
   WRONG:  title = comic.title + (comic.writer ? ` (${comic.writer})` : '')
   RIGHT:  title = comic.title

2. VOLUME DISPLAY — show volume as a separate badge, not merged into title
   The "VOL" column in the screenshots is correct — it shows volume as a number.
   Make sure your data fetch is NOT merging vol into the title string.
   
   In the JSON: each comic has:
     { "title": "All-New Ultimates", "issue": "#5", "volume": 1, ... }
   
   Display them separately:
     <Title>{comic.title}</Title>
     <Issue>{comic.issue}</Issue>
     <VolBadge>Vol {comic.volume}</VolBadge>

3. GROUPING LOGIC — when grouping by series, group on (title + volume) NOT just title
   This prevents Vol 1 and Vol 2 of the same title appearing as one run.
   
   Series key: `${comic.title} Vol ${comic.volume}`
   Display name: comic.title (with a "Vol 2" badge when volume > 1)

JSON DATA FIELDS:
-----------------
The compact JSON (marshall_comics_compact.json) has:
  comic.title      — clean title, no writer names, no (Vol X) suffix
  comic.issue      — issue number
  comic.volume     — integer (1, 2, 3...)  
  comic.year       — 4-digit year string
  comic.writer     — writer(s)
  comic.artist     — artist(s)
  comic.crossover  — crossover event name if applicable
  comic.isKey      — boolean
  comic.keyWhy     — string describing why it's a key
  comic.isSigned   — boolean
  comic.signedBy   — signer name(s)
  comic.box        — box number (integer)
  comic.era        — Golden/Silver/Bronze/Copper/Modern
  comic.publisher  — Marvel/DC/Image/etc

RUNS VIEW — correct grouping:
------------------------------
When building the "Runs" tab, group by:
  Key: title + volume number
  Sort: by issue number (numeric, not alphabetic)
  Label: show "Vol 2" badge only when volume > 1

This prevents "Moon Knight Vol 7 Bendis" and "Moon Knight Vol 9 MacKay" 
from appearing as a single run, and prevents issues appearing out of order.

PARSING LEGACY NUMBERS:

    legacyNum = 'Legacy ' + match[2];
  }
