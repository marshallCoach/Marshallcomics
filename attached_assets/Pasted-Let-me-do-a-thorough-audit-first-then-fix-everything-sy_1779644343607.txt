Volume — how it works now

Column 34 is the new Volume field. Every single book has a number. The logic:

	•	Explicit — 443 books where the volume was extractable from notes or arc data
	•	Year-derived — the majority, using a 60-rule lookup table (Batman 1940-2011 = Vol 1, Batman 2011-2016 = Vol 2 New 52, etc.)
	•	Default = 1 — for single-run titles or where the year falls outside any known volume range

This means Batman #656 (2006) correctly shows Vol 1, X-Men #1 (1991) shows Vol 2, Moon Knight #1 (2014) shows Vol 6 (the Ellis/Shalvey run), and so on.

Thumbnails — the coverThumbnails.js module

Drop this into Replit. It has three modes:

Mode 1 — Spine SVG (no API, instant): generateSpineSVG(comic) — generates a coloured vertical spine for the box view. Publisher-coloured gradient, gold cap on keys, green left stripe on signed books, red stripe on CGC priority. Title and issue number rendered vertically. This replaces the external links immediately with no API calls.

Mode 2 — Thumbnail SVG (no API, instant): generateThumbnailSVG(comic) — larger cover placeholder for the detail panel. Publisher branding, title, issue number, writer, year. Looks like a real cover in the right colour scheme.

Mode 3 — Real cover images (API, cached): getCoverUrl(comic) — fetches from Marvel API (Marvel books) or ComicVine (everything else). Falls back to Mode 2 if the API returns nothing or isn’t configured. Results cached in memory for the session.

For Replit, the quickest win is Mode 1 + Mode 2 — zero API keys needed, works immediately, looks great. Add the API keys later when you want real photographic covers on the keys.
